Section 01
Getting Started & Installation

The Learning Level Gap Solver is a Progressive Web App (PWA). It works instantly in any browser — no installation required. However, it can also be installed to the home screen for a native app-like experience.

1

Open in Browser

Open the index.html file in Chrome, Edge, or any modern browser. The app loads immediately — no internet connection required after the first load.

2

Install as App (Android / Desktop)

After a few seconds, an Install Banner will appear at the bottom of the screen. Tap "Install" to add the app to your home screen. It will then behave like a native Android app — offline, with its own icon and splash screen.

💡 On desktop Chrome: look for the install icon (⊕) in the address bar. On Android: tap the three-dot menu → "Add to Home Screen".
3

Offline Usage

Once opened once with internet (to load fonts), the entire app works fully offline. All progress, quiz scores, and settings are saved automatically to your device using browser storage. No account or login is needed.

Section 03
Diagnostic Assessment Guide

The Diagnostic Assessment is the most important entry point for new students. It determines the student's learning level — Foundation, Standard, or Advanced — and sets their personalised learning pathway. This process is aligned with the PARAKH assessment framework.

1

Fill in Student Information

Enter your Name, Class/Grade, Subject, and optionally your school name. All fields (except school) are required. This information is saved locally on your device.

2

Answer 10 Questions

The assessment contains 10 multiple-choice questions spanning three cognitive levels: foundational, standard, and advanced. Read each question carefully and select your best answer. There is no time limit — the assessment is not a race.

💡 Questions progress from easier to harder. Don't be discouraged if later questions are challenging — that's intentional and helps identify your level precisely.
3

After Each Answer

Immediately after selecting an answer, the correct answer is highlighted in green. If you chose incorrectly, your selection is shown in red and the correct answer is revealed. This immediate feedback supports active learning.

4

View Your Result

After Question 10, your diagnostic score and level placement are displayed. You will see your percentage score, the level you have been placed at, and a brief description of what this means for your learning journey. Tap "Start [Level] Learning" to go directly to lessons at your level.

✅ You can retake the assessment anytime. If you feel your level placement does not reflect your actual ability, tap "Retake Assessment." There is no limit on retakes — consistent with NCF 2023's growth-oriented assessment philosophy.

Section 05
Using Lessons Effectively

When you tap a topic card, the Lesson Viewer opens below the topic grid. Each lesson is structured in a consistent, NCF 2023-aligned format:

1

Read the Introduction

Every lesson opens with a clear introduction explaining what the topic is about and why it matters. Foundation Level introductions use simple language and visual metaphors. Standard Level uses curriculum terminology. Advanced Level uses research-level vocabulary.

2

Study the Concept Box

Blue Concept Boxes highlight the key formula, equation, or concept statement for the topic. This is the essential takeaway — the single most important thing to remember. For Science, this is often a chemical equation; for Maths, a formula; for Social Science, a key principle.

3

Understand the Visual Diagram

Many lessons include a Visual Diagram — a simplified illustration showing the process or relationship being described. Study this carefully; research shows that visual-verbal dual coding significantly improves retention (Mayer's Multimedia Learning Theory).

4

Answer the Practice Question

Every lesson ends with a Practice Question. Select your answer — feedback appears immediately. If wrong, the correct answer is highlighted. This is not graded; it is purely for self-assessment and reinforcement.

💡 If you get the practice question wrong, re-read the concept box before moving to the next topic. True mastery requires returning to misunderstood points.
5

Mark Complete

After reviewing the lesson, tap "✓ Mark Complete" to record progress. This updates your dashboard statistics, may unlock achievement badges, and marks the topic card with a ✅ checkmark.

Section 06
Simulation & Activity Guide

Several topics include simulation-style visual content — especially Science topics at Standard and Advanced levels. These are not full interactive simulations (which require internet), but guided conceptual diagrams and step-by-step visual walkthroughs that simulate the process.

How to Use Simulation Content

1
Open the relevant topic (e.g., "Photosynthesis" at Standard Level)
2
Read the introduction — understand the overall process before examining the diagram
3
Study the Visual Diagram section — trace the process visually (e.g., CO₂ + H₂O → Glucose + O₂)
4
Read the Concept Box — connect the visual to the formal scientific/mathematical representation
5
Apply learning to the Practice Question — this tests whether you can transfer understanding to a novel question
6
At Advanced Level: re-read the analytical section and consider why different pathways (C3/C4/CAM) exist — this develops scientific reasoning
⚠️ Simulation Tips for Teachers: When using the app as a classroom tool, display the diagram section on a projector and ask students to predict the next step before revealing it. This Socratic approach deepens engagement with simulated content.

Section 08
Teacher Dashboard Guide

The Teacher Dashboard is designed for classroom educators. It provides analytical insights into learning level distribution and tools for targeted intervention — aligned with NCF 2023's vision of the teacher as learning facilitator.

👨‍🏫 Teacher Dashboard
📊
Class Distribution Chart: A bar chart showing the percentage of students at Foundation, Standard, and Advanced levels. Updated from student diagnostic data. Use this to plan your differentiated instruction sessions.
📋
Student Roster: A tabular view of all students with their names, level placements, diagnostic scores, and activity status. Tap "Export" to generate a printable roster (simulated in demo; full export in school LMS integration).
⚡
Quick Action Cards:
📝 Assign Foundation Task — sends targeted exercises to Foundation Level learners
🚀 Challenge Advanced — sends HOTS tasks to Advanced Level students
📊 Generate Report — creates a PARAKH-format progress summary
🔔 Send Reminder — notifies inactive students
📜
NEP 2020 Insight Panel: A contextual reminder of the policy framework the dashboard operates within — helping teachers frame their data in the broader NEP and NCF mandate.

Classroom Usage Workflow

1

Monday: Review Distribution

Open the Teacher Dashboard at the start of the week. Check how many students are at each level. Plan three differentiated activity sets for the coming week's topic.

2

During Class: Parallel Learning

Direct Foundation students to open the app and work through Foundation Level content. Standard students work through Standard Level. Advanced students tackle the Advanced Level — potentially after completing Standard Level work.

3

After Class: Check Roster

Review the roster for students marked "Inactive." These are students who have not engaged. Use the "Send Reminder" action or follow up personally.

4

Monthly: Generate Report

Use the Generate Report function to create a PARAKH-aligned progress summary for school administration or parent meetings. This documents the differentiated instruction approach for official records.

Section 10
Technical Setup & Hosting

📁 File Structure

llg-app/ ├── index.html ← Main application (all-in-one) ├── manifest.json ← PWA configuration ├── service-worker.js ← Offline caching ├── author.jpg ← Author photo ├── conceptual-background.html ← This theory document └── user-manual.html ← This manual

🌐 Running Locally

1

Simple Method — Open Directly

Double-click index.html to open in browser. Note: Service Worker may not work in this mode, but the app itself functions fully.

2

Recommended — VS Code Live Server

Install VS Code → Install the "Live Server" extension → Right-click index.html → "Open with Live Server". This enables full PWA features including Service Worker and offline mode.

🚀 Free Hosting Options

A

GitHub Pages (Recommended)

1. Create a free GitHub account → Create a new repository → Upload all files → Go to Settings → Pages → Deploy from main branch. Your app will be live at https://yourusername.github.io/repo-name.

B

Netlify (Easiest)

Go to netlify.com → Drag and drop your entire project folder onto the deploy zone. App is live within 30 seconds with a free URL. Netlify automatically handles HTTPS and PWA features.

C

Cloudflare Pages

Free, fast, global CDN hosting. Connect your GitHub repository and Cloudflare automatically deploys on every update.

📱 Converting to Android APK

1

Using PWABuilder (Easiest)

Go to pwabuilder.com → Enter your hosted app URL → Click "Start" → Select Android → Download the APK package. This creates a Trusted Web Activity (TWA) wrapper — the same technology used by major apps.

2

Upload to Google Play Store

1. Create a Google Play Console account ($25 one-time fee) → Create new app → Upload your APK/AAB → Complete store listing with screenshots, description, category → Submit for review. Review typically takes 3–7 days for educational apps.

✅ For school deployment: The simplest method is to host on GitHub Pages for free, share the URL with students, and have them add to home screen. No APK, no Play Store, no cost. Works identically to a native app.
